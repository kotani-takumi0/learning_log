# Learning Log UI 実装タスク

- [ ] 1. プロジェクト基盤のセットアップ  
  説明: Vite + React + TypeScript プロジェクトの構成を確認し、`src` 配下にアプリ本体・コンポーネント・スタイルを配置するための基本ディレクトリを整える。CSS Modules を利用できる状態にする。  
  完了条件: `npm run dev` で起動でき、CSS Modules ファイルを読み込める最小構成が用意されている。

- [ ] 2. 学習記録データモデルと永続化ユーティリティの作成  
  説明: `LearningLog` インターフェース（`id`, `content`, `time`）を定義し、LocalStorage への保存・読込を行うユーティリティ関数を実装する。  
  完了条件: `LearningLog[]` を LocalStorage に保存し、再取得できる関数が用意されている。

- [ ] 3. 学習記録入力フォームの実装  
  説明: 学習内容テキスト入力、学習時間数値入力、追加ボタンを持つフォームを実装し、入力値を親コンポーネントへ渡せるようにする。  
  完了条件: 2つの入力項目と追加ボタンが表示され、ボタン操作で入力値を受け取る処理が呼び出される。

- [ ] 4. 記録追加ロジックと状態管理の実装  
  説明: 追加操作時に `id` を採番して `LearningLog` を生成し、React の state に反映する。最新記録が先頭になるよう新しい順で配列を更新する。  
  完了条件: 追加するたびに state が更新され、直近で追加した記録が先頭に配置される。

- [ ] 5. 学習記録一覧表示コンポーネントの実装  
  説明: state 内の学習記録をリスト形式で描画するコンポーネントを作成し、内容と学習時間を見やすく表示する。  
  完了条件: 追加済みの記録が一覧で表示され、表示順が新しい順になっている。

- [ ] 6. LocalStorage 連携のアプリ統合  
  説明: 初回表示時に LocalStorage から記録を読み込み、記録追加時に最新 state を LocalStorage へ保存する処理を `App` に統合する。  
  完了条件: ブラウザをリロード・再起動しても記録が保持され、画面表示に復元される。

- [ ] 7. CSS Modules による画面スタイリング  
  説明: フォームとリストのレイアウト、余白、視認性を CSS Modules で整え、学習記録メモ帳として使いやすい見た目にする。  
  完了条件: 入力エリアと一覧が崩れず表示され、スタイルがコンポーネント単位で分離されている。

- [ ] 8. 動作確認と受け入れチェック  
  説明: 入力、追加、新しい順表示、LocalStorage 永続化の要件を一通り手動で確認し、要件定義との整合をチェックする。  
  完了条件: 要件定義書の機能要件（入力・表示・保存）をすべて満たすことを確認できる。
